# Отчет

## Принцип работы

Алгоритм КМП выполняет поиск подстроки за линейное время, исключая повторные проверки символов в тексте. Основная особенность: указатель прохода по тексту движется только вперед.

Алгоритм состоит из двух этапов:

### 1. Построение массива LPS (Longest Prefix Suffix)
Перед поиском выполняется препроцессинг паттерна. Строится массив `LPS`, где значение `LPS[i]` — это длина самого длинного **префикса** (начала) подстроки `pattern[0...i]`, который одновременно является её **суффиксом**.

Это значение показывает, какую часть паттерна можно не проверять заново при несовпадении, так как эта часть уже гарантированно совпадает с пройденным участком текста.

### 2. Этап поиска
Используются два индекса: `i` для текста и `j` для паттерна. Сравнение идет посимвольно.

*   **При совпадении (`text[i] == pattern[j]`):**
    Оба индекса увеличиваются.

*   **При несовпадении (`text[i] != pattern[j]`):**
    *   Индекс текста `i` **не меняется** (не откатывается назад).
    *   Индекс паттерна `j` обновляется значением из таблицы: `j = LPS[j-1]`.
    *   Это действие сдвигает паттерн вправо относительно текста так, чтобы выровнять уже найденный совпадающий суффикс с идентичным префиксом паттерна. Проверка продолжается с текущей позиции `i`.

## Вычислительная сложность

Пусть **N** — длина текста, **M** — длина паттерна.

1.  **Построение LPS:** Требует одного прохода по паттерну — **O(M)**.
2.  **Поиск:** Указатель `i` проходит по тексту от 0 до N, никогда не уменьшаясь. Указатель `j` может возвращаться назад, но суммарное количество его сдвигов не превышает N — **O(N)**.

**Итоговая сложность:** **O(N + M)**.
